Tracking
❓ Lines link receivers that measure each other. Colour = distance error (see legend); grey dashed = moved, recalibrate. Hover a line or receiver to read its detected (real) distance. The selectors limit the count, filter by colour, and switch the detected distance between calibrated and raw. Click a line/pill to isolate, a receiver for its links. ❓ Show each receiver's measured distance to the tracked beacons, with the value on top of each receiver. With one beacon shown, each receiver draws a full distance circle (its own colour) with a thick line to the beacon inside it — where the circles intersect is the beacon. With several beacons the circles would overlap into noise, so each beacon instead shows thin lines to the receivers hearing it, in that beacon's colour. Click a beacon (on the map or in the list above) to isolate it. ❓ Draw the path each tracked device has taken since tracking started, in that device's colour — brighter is more recent. The traces clear when a new tracking session starts.

Receiver Calibration

❓ Compares the distances receivers measure between each other (each probe advertises an iBeacon) with their placed positions, and fits a per-receiver distance correction. Select the floor first, then sample while the house is quiet.
❓ Continuously samples in the background, re-solves every 15 minutes for every floor, and re-applies corrections whenever they change. Survives Home Assistant restarts.

Receiver linking

❓ How each placed receiver maps to its Bermuda distance sensors, and whether each is reporting a distance right now. Use this to tell a real naming mismatch (Unmatched) apart from a receiver that is linked correctly but simply not reporting at the moment (No reading). This is a live snapshot — press Refresh to re-check.
